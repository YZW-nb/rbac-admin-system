## 一、项目概述

### 1.1 项目简介

一个基于 RBAC（Role-Based Access Control）模型的企业级权限管理系统，实现用户、角色、菜单、部门、日志的完整管理，支持按钮级权限控制。

### 1.2 技术栈

| 层次 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Spring Boot | 3.2.x |
| 持久层 | MyBatis-Plus | 3.5.x |
| 数据库 | PostgreSQL | 15+ |
| 缓存 | Redis | 7.x |
| 认证 | JWT（jjwt） | 0.12.x |
| 接口文档 | Knife4j（Swagger 增强） | 4.x |
| 分页 | PageHelper | 2.x |
| 前端框架 | Vue 3 | 3.4.x |
| UI 组件库 | Element Plus | 2.x |
| 状态管理 | Pinia | 2.x |
| 路由 | Vue Router | 4.x |
| HTTP 客户端 | Axios | 1.x |
| 构建工具 | Vite | 5.x |
| Java 版本 | JDK | 17+ |

## 五、核心流程设计

### 5.1 登录认证流程

```
用户输入账号密码
       │
       ▼
┌──────────────┐
│ 查询用户信息   │
│ 验证密码(BCrypt)│
└──────┬───────┘
       │ 验证通过
       ▼
┌──────────────┐     ┌──────────────┐
│ 生成 JWT     │────▶│ 存入 Redis    │
│ (access+refresh)│   │ 设置过期时间   │
└──────┬───────┘     └──────────────┘
       │
       ▼
┌──────────────┐
│ 返回 Token   │
│ + 用户基本信息 │
└──────────────┘

=== 后续请求 ===

请求携带 Authorization: Bearer <token>
       │
       ▼
┌──────────────────────┐
│ JwtAuthenticationFilter│
│ 1. 提取 Token          │
│ 2. 验证签名 + 过期时间  │
│ 3. 检查 Redis 是否有效  │
│ 4. 设置 SecurityContext │
└──────────┬───────────┘
           │
           ▼
      放行到 Controller
```

### 5.2 动态权限加载流程

```
登录成功后
     │
     ▼
┌──────────────────────┐
│ 根据用户ID查询角色     │
│ 根据角色ID查询菜单权限  │
└──────┬───────────────┘
       │
       ▼
┌──────────────────────┐
│ 构建菜单树（过滤按钮） │
│ 缓存到 Redis          │
└──────┬───────────────┘
       │
       ▼
  返回前端路由数据

=== 前端处理 ===
     │
     ▼
┌──────────────────────┐
│ 解析路由数据           │
│ 动态注册 Vue Router   │
│ 渲染侧边栏菜单        │
│ 存储权限标识列表       │
└──────┬───────────────┘
       │
       ▼
┌──────────────────────┐
│ v-permission 指令     │
│ 按钮级别权限控制       │
└──────────────────────┘
```

### 5.3 AOP 操作日志流程

```
Controller 方法调用
       │
       ▼
┌──────────────────────┐
│ LogAspect 切面拦截     │
│ @Log(title="用户管理", │
│      businessType=INSERT)│
└──────┬───────────────┘
       │
       ▼
┌──────────────────────┐
│ 环绕通知：            │
│ 1. 记录请求参数        │
│ 2. 放行目标方法        │
│ 3. 记录响应结果        │
│ 4. 记录执行耗时        │
│ 5. 异常时记录错误信息   │
│ 6. 异步写入日志表       │
└──────────────────────┘
```

## 六、Redis 缓存设计

### 6.1 缓存 Key 规范

```
格式：项目名:模块:业务key
示例：admin:login:token:uuid
      admin:user:info:userId
      admin:menu:role:roleId
```
